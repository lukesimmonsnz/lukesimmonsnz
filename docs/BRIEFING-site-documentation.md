## 1. What this project is

A local-first personal website. One Flask app, one hand-written CSS file, no JavaScript framework, no analytics, no cloud. It runs from the owner's machine via `start.bat` / `stop.bat`. The site has several distinct sections, each implemented as a Flask blueprint:

- A **home / projects / now / contact** section (main).
- A **blog** reading Markdown posts with YAML frontmatter.
- A **research** collection with three hand-written branches (Computer Science, Climate Science & AI, Medical Science) and one data-driven branch — **Auckland**.
- A **David Simmons biography** section (the owner's grandfather, an ethnologist).

The most technically interesting part is the **Auckland research project**: a content-as-data knowledge base rendered from a typed entity graph (schemas + YAML + lint + render pipeline). Treat it as a first-class subsystem in the technical doc, because it is architecturally distinct from the rest of the site.

## 4. Auckland subsystem — concrete details

Because this is the least standard part of the site, here are concrete facts the new session needs without having to re-derive them:

- **Schemas**: `content/auckland/schema/*.schema.json`. Eight schemas: `problem`, `evidence`, `driver`, `response`, `camp`, `source`, `metric`, `actor`. JSON Schema Draft 2020-12. Each schema uses `additionalProperties: false` and enforces a type-prefixed id pattern (e.g. `^camp\.[a-z0-9_.]+$`).
- **Data**: `content/auckland/data/<type>/<slug>.yaml`, one entity per file. Current entity count is around 69 (verify with `python tools/graph.py` from `content/auckland/`).
- **Templates**: `content/auckland/templates/subpage.md.j2` is the sole template. It renders Markdown with YAML frontmatter.
- **Tools**: `content/auckland/tools/graph.py`, `lint.py`, `render.py`. Entry points:
  - `python tools/graph.py` — dump a graph summary; exits non-zero on schema errors.
  - `python tools/lint.py` — run graph invariants; exits non-zero on any failure.
  - `python tools/render.py <problem.id>` or `python tools/render.py --all` — generate Markdown under `content/auckland/pages/`. Always lints first; refuses to render on lint failure unless `--skip-lint` is passed.
- **Briefings directory**: `content/auckland/briefings/*.md`. **Private decision-support documents authored interactively between Claude and the owner. Do not publish, do not summarise in public docs, do not mention their existence in the README or sitemap.**
- **Generated pages**: `content/auckland/pages/**/*.md`. These are written by the renderer; never hand-edit. The Auckland blueprint reads them and serves them at `/research/auckland/<section>/<subpage>/`.
